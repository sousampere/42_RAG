## ℹ️ 1. Project Overview

### What the subject is about

The goal of the project is to :

- **<ins>Ingest</ins>** : Create a searchable knowledge base like google

    -> Read and process files from the vLLM repo

    -> Chunk data in different ways for Python code and Markdown documentation

    -> Create a searchable index using TF-IDF or BM25
    
    -> Store these index for fast retrieval

- **<ins>Search</ins>** : Be able to type a question and get relevant code snippets / documentation for this question, using this knowledge base

    -> Implement semantic search over the indexed knowledge base

    -> Return top-k most relevant result for any query (including `file_path`, `first_character_index`, `last_character_index`)

    -> Support a batch processing of multiple questions from a JSON dataset

    -> Achieve at least 80% recall@5 on docs questions, and 50% on code questions

- **<ins>Answer</ins>** : Answer the questions using an LLM (Qwen/Qwen3-0.6B) with the given documentation / code snippets

    -> Use Qwen/Qwen3-0.6B to answer questions with natural language

    -> Give the retreived context to the LLM, within its context window

    -> Generate answers based on the retrieved code / documentation

    -> Output structured JSON following provided models

- **<ins>Evaluate</ins>** : Evaluate the retrieval system quality using recall@k metrics

    -> Implement recall@k metric to measure retrieval quality

    -> Compare retieved sources against ground truth annotations

    -> Calculate overlap between retrieved and correct sources (min 5% overlap) : The output source must cover at least 5% of the true source

    -> Provide detailed performances metrics

- **<ins>CLI</ins>** : All the actions will be triggered by CLI commands

    -> Include progress bars for long-running operations

    -> Clear errors messages, no tracebacks, handle edge cases

## 🔧 3. Program structure

### Makefile

Creation of a structured makefile with overridable variables.

### CLI

Creation of the CLI using Python Fire.

### Indexing

Create a custom `BM25sRetriever` retreiver class for langchain, inheriting from `BaseRetriever` that can index the knowledge base, save it and load it again.

Get max_chunk_size -> gather raw files to get a list of .md and .py files -> load file content -> create chunks -> tokenize chunks -> index / score -> save chunks

### Searching

Use the created/loaded retriever with the `retriever.invoke("texte")` to search a term in the database.

Use the same principe on multiple strings to create multiple requests.

### Answering

Use the searching tool and connect an LLM to it to give answers.

### Evaluation

Calculate the overlap between the obtained source, and the true one, and check if at least 5% of the obtained source is in the real one. Repeat this for every prompt, and calculate the average success probability for docs and code.
